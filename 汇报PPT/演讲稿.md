# 《分层证据驱动的视频生成质量评估框架》演讲稿

## 1. 封面

各位老师、同学好，我们小组今天汇报的题目是《分层证据驱动的视频生成质量评估框架》。

这个项目关注的是 AI 生成视频的质量评估问题。我们并不只是判断一个视频“好不好看”，而是希望回答三个更具体的问题：视频中到底哪里出现了错误，这类错误应该归入哪个评价标签，以及这个标签背后的证据能不能被人工复核。

何嘉伟：负责项目总体框架设计与流程统筹，包括任务背景梳理、技术路线设计、三层证据链方案构建，以及汇报材料整体逻辑整合。
杨浩：负责视频解析流程搭建，包括宏观层、片段层、逐帧层 Prompt 设计与迭代，整理视频分析结果，并优化提示词要素核对机制。
张智强：负责实验数据整理与标签体系与裁判模型设计，包括标签定义整理、正反例补充、裁判 Prompt 编写、分批打标逻辑设计，以及最终标签结果校验。
郑渡元：负责可视化展示，包括样本视频生成、分析结果归档、准确率与召回率看板制作、PPT 页面设计和演讲稿整理。

因此，我们采用的总体思路是“先描述，后裁判”。也就是说，先让多模态模型把视频转化为不同层级的事实描述，再让裁判模型基于原始 Prompt、标签体系和这些事实证据完成结构化打标。这样可以把原本比较黑箱的视频评价过程，转化为一条更清晰、更可追溯的证据链。

## 2. 研究背景与问题定义

首先来看研究背景。

在 AI 视频生成场景中，人能够综合判断主体、动作、场景、风格、音频和整体观感。但是，一旦视频数量上升，人工标注和复核的成本会迅速增加，而且在团队作业项目中，标注口径和一致性也很难长期保持一致，甚至会出现逆序的情况。

如果直接把视频交给多模态大模型，让模型端到端输出标签，也会遇到新的问题。模型可能在证据不足时产生幻觉，也可能给出一个看似合理的标签，却没有说明这个标签来自哪一帧、哪一段、哪一个可观察细节。对于质量评估来说，只有结论是不够的，结论必须能够被追溯和复核。

所以，本项目把问题重新定义为：如何把复杂的视频质量判断，转化为可审计、可追溯、可复核的文本证据判定。我们不是让模型直接替代人工判断，而是让模型先形成结构化证据，再在证据基础上辅助完成标签归因。

## 3. 多模态视频解析的技术约束

这一页说明为什么视频评估不能简单地理解为“把视频丢给模型就可以”。

多模态大模型进行视频解析时，通常不会完整读取视频的每一帧，而是先进行抽帧或片段采样，再由视觉编码器提取视觉特征，并将这些特征压缩成语言模型可以处理的视觉 token。也就是说，视频在进入语言模型之前，已经经历了采样、压缩和语义对齐。

这会带来三个限制。

第一是抽帧限制。视频中的短暂错误，比如手部一帧畸变、文字瞬时错误、物体短暂穿模，可能刚好没有被采样到。

第二是表征压缩。连续画面被压缩为有限数量的视觉 token 后，细粒度文字、局部结构和微小动作容易被弱化。

第三是语义幻觉。语言模型可能根据常识补全不存在的对象、动作或场景。如果这些幻觉进入后续裁判环节，就会造成错误打标。

因此，本项目没有采用“视频输入、标签输出”的端到端方式，而是在中间增加了分层证据环节，让模型先记录事实，再进行归因。

## 4. 系统流程

整个系统流程可以概括为五个步骤。

第一步是原始输入。输入内容包括待评估视频、生成该视频的 Prompt、标签体系，以及宏观层、片段层、逐帧层和裁判层的提示词文档。

第二步是三层描述。宏观层、片段层和逐帧层分别从不同粒度描述视频内容，但它们都不直接输出标签，也不判断对错。它们只负责把可以观察到的事实讲清楚。

第三步是分批裁判。裁判模型不会一次性处理所有标签，而是按照一级标签分批判断，每次只关注一个一级标签下的二级标签。这样可以降低上下文压力，也能让模型更专注于当前标签边界。

第四步是去重归因。后续批次会携带当前已有标签结果，判断新的错因是否已经被覆盖，是替换、保留，还是少量并列。这里的原则是尽量避免同一个错误被反复归因。

第五步是最终 JSON 输出。最终结果只保留三个核心字段：“一级标签--二级标签”“证据来源”和“错因描述”。这样既便于人工查看，也便于后续统计准确率、召回率和标签分布。

## 5. 三层证据权责

三层证据是本项目最核心的设计。

宏观层负责全片范围内稳定成立的事实，例如主体是谁、整体场景是什么、视觉风格是否稳定、全局事件是否清楚、整体音频是否存在明显问题。它更关注视频的总体状态。

片段层负责时间过程中的变化，例如不同时间段发生了什么、动作是否连续、主体和物体是否发生变化、事件顺序是否合理、音画关系是否一致。它更关注视频在时间轴上的过程。

逐帧层负责局部可见细节，例如单帧中的主体结构、画面文字、局部畸变、清晰度、构图、色彩，以及相邻帧之间是否出现突变。它更适合捕捉细粒度画面问题。

这里需要强调的是，描述层不可见标签，也不承担判断责任。它们不会说“这是主体错误”或“这是动作错误”，而只会说视频中看到了什么、没看到什么、哪里不可见、哪里不确定。标签判断由后续裁判层完成。

这种分工可以减少分析层被标签名误导的风险，也让后续裁判能够基于更干净的事实证据进行判断。

## 6. 标签体系

当前标签体系包含 7 个一级标签和 31 个二级标签。

提示词一致性关注生成视频是否符合 Prompt 要求，包括主体、属性、动作、场景和风格。

视觉生成质量关注画面本身的生成问题，包括清晰度、结构畸变、局部生成失败和视觉伪影。

时间一致性关注视频在时间轴上的连续性，包括角色突变、物体突变、场景突变、动作断裂和事件跳变。

物理真实性关注运动和交互是否符合基本物理规律，包括人体运动、物体运动、碰撞接触和自然现象。

音频生成质量关注台词、音画同步、音频失真、情感表达和环境音。

审美质量关注构图、镜头设计、色彩和节奏。

安全与合规关注暴力、色情、隐私和版权风险。

为了降低标签混淆，每个标签不仅有定义，也需要配套正例和反例。正例帮助模型理解标签适用边界，反例则帮助模型区分相近但不应归入该标签的情况。

## 7. 样本视频

这一页展示的是当前用于汇报的样本视频。

样本 Prompt 要求生成一段 8 秒的真人古装写实短片。核心要素包括：清晨薄雾中的古代庭院，一名年轻女书生，浅青色交领长衫，木簪束发；中段需要拿起淡黄色竹简，左手按住毛笔，并且竹简上要出现清晰黑色文字“古风测试”；后段还要求水缸倒影、竹叶落下、动作连续和画面清晰。

在我们的流程中，视频不会直接进入标签判断，而是先进入三层描述。模型需要围绕 Prompt 中的核心要素记录事实状态，例如是否出现指定主体、指定服装、指定文字、指定动作和指定场景。这样后续裁判才有依据判断视频是否偏离 Prompt。

## 8. 证据抽取：提示词要素核对

这一页对应我们在开发过程中做的一项重要改进。

早期版本中，三层描述虽然会概括视频内容，但有时不会覆盖 Prompt 里的关键要求。比如 Prompt 明确要求一个主体“无嘴巴”，如果描述中没有提到是否有嘴巴，后续裁判就可能因为证据缺失而误判。

因此，我们在三层描述提示词中加入了“提示词要素核对”机制。模型需要先从 Prompt 中提取核心要素，再在描述中记录这些要素的状态，例如“有”“无”“不可见”或“未知”。它仍然不判断对错，但必须把和 Prompt 相关的事实讲清楚。

在当前样本中，宏观层会确认古代庭院、服装、竹简等全局要素，同时指出是否出现指定文字。片段层会按时间段描述动作和场景变化。逐帧层会进一步记录文字是否清晰、主体结构是否完整、发型或服装是否突变等细节。

这样做的意义是把 Prompt 中的要求显式转化为可检查的证据点，减少“没有描述，所以无法判断”的情况。

## 9. 三层描述展示

这一页展示的是三层描述的实际输出。

宏观层像一份全片摘要，帮助我们快速知道视频整体呈现是否与 Prompt 大方向一致。它关注稳定事实，而不是瞬间细节。

片段层更像时间轴记录，会说明 0 到 3 秒、3 到 5 秒、5 到 8 秒分别发生了什么。通过片段层，我们可以观察动作是否按要求发生、主体是否持续存在、场景是否突然变化。

逐帧层则更接近证据截图说明。它会把关键帧里的局部结构、文字可读性、清晰度、构图和相邻帧变化记录下来。对于“文字没有生成清楚”“发型突然变化”“主体被裁切”这类问题，逐帧层通常能提供更直接的证据。

三层描述不是互相替代，而是互相补充。宏观层保证全局覆盖，片段层保证过程覆盖，逐帧层保证细节覆盖。

## 10. 裁判输出：最终标签与错因

这一页展示的是裁判模型的最终输出形式。

最终结果不再保留冗长的推理过程，而只展示三个字段：第一是“一级标签--二级标签”，第二是“证据来源”，第三是“错因描述”。

例如，如果逐帧层指出竹简上的文字模糊不可读，且 Prompt 要求出现清晰黑色文字“古风测试”，裁判层就可以将其归入提示词一致性下的相关标签，并说明证据来自逐帧层的对应帧。

如果片段层显示人物没有完成指定动作，裁判层则可以将其归入动作错误，而不是笼统地归入主体错误或属性错误。

这种输出形式的优点是简洁、可复核、可统计。人工复审时可以直接根据证据来源回看对应描述；系统迭代时也可以统计哪些标签准确率低、哪些标签召回率不足、哪些标签存在重复归因。

## 11. 开发过程问题与改进

在开发过程中，我们遇到的一个典型问题是标签归因错误。

具体来说，模型有时已经发现视频中存在问题，但会把问题归入不够准确的标签。例如，视频中的人物没有执行 Prompt 指定动作，模型能够发现“动作不符合要求”，但可能错误归因到“人物属性错误”。这说明问题并不完全在于模型看不见错误，也在于模型不一定能稳定区分相近标签。

针对这个问题，我们做了三类改进。

第一，在标签定义中加入正例和反例，让模型看到标签边界。

第二，在裁判提示词中要求模型先描述错因，再选择标签，避免直接按标签名做表面匹配。

第三，在分批裁判时携带已有打标结果，让模型判断当前错因是否已经被覆盖，减少同一问题被重复打标。

这些改进不能完全消除误判，但可以让错误更容易定位，也让后续迭代更有方向。

## 12. 当前项目的核心局限

虽然当前链路已经形成了可运行闭环，但它仍然有几个核心局限。

第一，主观维度仍然较弱。审美质量、情感表达、节奏感、氛围感这些标签，本身就高度依赖评价者偏好和具体任务语境。仅靠提示词工程，很难让模型稳定达到一致判断。后续需要更细粒度的评价量表，或者引入人工标注数据进行校准。

第二，细粒度画面问题仍可能漏检。逐帧层并不等于真正逐帧穷举，它依赖模型或接口的采样能力。手部瞬时畸变、文字一帧错误、短暂穿模等问题，仍可能因为没有被采样到而漏掉。后续可以考虑自适应采样，也就是优先关注画面骤变帧、不连贯帧、运动异常帧和低置信片段。

第三，证据质量会向后传导。裁判层依赖三层文字证据，如果分析层遗漏了关键信息，或者描述中出现幻觉，后续标签判断也会受到影响。因此，系统不仅要优化裁判提示词，也要持续优化前端证据生成质量。

所以，后续方向可以概括为四点：主观量表细化、自适应采样、低置信证据回填，以及人工复核闭环。

## 13. 问题解决前后对比

这一页可以用来概括项目方法的变化。

在问题解决前，模型更容易直接从视频或描述跳到标签结论，中间缺少明确证据链。标签之间的边界也容易混淆，同一错因可能被重复归入多个标签。

在问题解决后，系统把评估流程拆成两个阶段：先由三层描述生成事实证据，再由裁判模型结合 Prompt、标签定义和已有标签结果进行归因。

这个变化的关键意义在于：错误不再只是一个抽象标签，而是可以追溯到具体层级、具体片段或具体帧。对于人工复审来说，这比单纯得到一个标签更有价值；对于系统优化来说，也更容易知道问题出在视频解析、证据描述，还是标签裁判。

## 14. 结尾

最后总结一下。

本项目的核心价值，是把 AI 生成视频中的质量问题，从一个不可解释的端到端判断，转化为一条可追溯的证据链。

在这条链路中，描述层负责事实，避免提前带入标签结论；裁判层负责归因，结合 Prompt、标签定义和三层证据完成结构化打标；数据看板负责迭代，持续分析准确率、召回率和重复归因问题。

从当前样本来看，这套流程已经能够完成从视频解析、事实描述、标签裁判到最终错因输出的闭环。它的重点不是宣称模型可以完全替代人工，而是让模型判断更可解释、更可复核，也更适合在批量评估场景中辅助人工工作。

后续如果进一步加入自适应采样、主观评价量表和人工复核机制，系统的稳定性和迁移性还可以继续提升。
