# 分层证据驱动的视频生成质量评估框架

## 1. 任务背景与研究动机

本项目旨在构建一套面向 AI 生成视频的视频质量评估流程，用于识别视频生成结果中可能存在的内容偏差、视觉缺陷、时间一致性问题、物理真实性问题、音频质量问题、审美问题以及安全合规风险。

在实际评估场景中，纯人工评价虽然具有较高可信度，但存在成本高、效率低、规模化困难等问题。当待评估视频数量增加时，人工标注的时间成本和一致性维护成本会迅速上升。

直接使用大模型对视频进行端到端评价同样存在明显局限。第一，大模型在复杂视频任务中容易产生幻觉，可能在证据不足的情况下直接给出判断。第二，端到端评价的可追溯性较弱，难以解释某一标签对应的具体视觉或音频依据。第三，视频质量评价本身包含主体、属性、动作、场景、时间连续性、物理规律、音频、审美、安全等多个维度，任务复杂度较高，直接让模型一次性完成全部判断容易造成遗漏、误判或标签混淆。

因此，本项目采用“先描述、后裁判”的分层证据驱动思路：先将视频内容拆解为多层文字化证据，再由裁判模型基于原始提示词、分层描述和标签体系进行打标。该方法本质上是对视频评估任务进行降维处理，将复杂的多模态判断问题转化为较可控的文本证据判定问题。

### 1.1 多模态大模型进行视频解析的基本技术原理

多模态大模型进行视频解析，通常不是直接在原始视频流上完成端到端语义判断，而是先将视频转化为可被语言模型处理的视觉 token 或视觉特征序列。典型流程包括：对视频进行抽帧或片段采样，使用视觉编码器提取帧级或时序视觉特征，再通过投影层、交叉注意力模块或统一表征对齐机制，将视觉特征映射到大语言模型可以理解的语义空间中。随后，大语言模型基于这些视觉表征和文本指令生成回答、摘要、描述或判断。

早期代表性多模态模型 Flamingo 通过连接预训练视觉模型和语言模型，使模型能够处理图文交错输入，并支持图像和视频输入下的少样本学习（Alayrac et al., 2022）。这类方法说明了一个重要趋势：视频理解并非单纯的视觉识别任务，而是需要将视觉表征与语言推理能力结合起来。

面向视频对话和视频理解任务，Video-ChatGPT 将视频适配的视觉编码器与大语言模型结合，使模型能够围绕视频内容进行细粒度对话，并构建了视频指令数据和定量评估框架（Maaz et al., 2023/2024）。这类模型通常依赖抽帧、时序特征聚合和语言模型推理来完成视频问答与视频描述。

Video-LLaVA 进一步讨论了图像和视频表征在进入语言模型前的对齐问题。该工作指出，如果图像和视频被编码到不同特征空间，再依赖少量投影层进行对齐，语言模型较难学习稳定的多模态交互；因此其提出在投影前进行统一视觉表征学习，以提升图像和视频任务之间的联合理解能力（Lin et al., 2023）。这说明视频解析效果很大程度上取决于视觉表征、时序信息和语言空间之间的对齐质量。

### 1.2 多模态视频解析在评估任务中的主要缺点

虽然多模态大模型已经能够生成较流畅的视频描述和问答结果，但将其直接用于视频生成质量评估仍然存在若干结构性缺陷。

第一，视频大模型容易受到抽帧和上下文压缩的限制。视频包含连续时间信息，但模型通常只能处理有限数量的帧或有限长度的视觉 token。抽帧策略如果没有覆盖关键时刻，模型就可能漏掉短暂出现的错误、局部结构异常、动作断裂或音画不同步等问题。MVBench 指出，许多已有多模态评测偏重静态图像式空间理解，而动态视频任务需要更系统的时间理解能力；其实验也表明，当任务需要从感知到认知层面的时序能力时，现有多模态大模型仍远未达到充分可靠的水平（Li et al., 2023）。

第二，视频理解模型在长视频和多模态信号整合方面仍然不足。Video-MME 从视频类型、视频时长、字幕和音频等多模态输入角度构建评测，指出多模态大模型在处理长时序、多场景和多模态数据时仍需要进一步提升（Fu et al., 2024）。这与本项目场景高度相关：视频生成质量评估不仅需要识别单帧画面是否合理，还需要判断主体是否持续一致、动作是否连续、事件是否符合因果顺序、音频是否与画面同步。

第三，多模态大模型存在幻觉问题。POPE 对大视觉语言模型的物体幻觉进行了系统研究，指出模型可能生成与图像内容不一致的物体描述，并且视觉指令中的高频对象或共现对象会影响幻觉倾向（Li et al., 2023）。在视频质量评估中，这种问题会进一步放大：如果模型在没有充分证据时生成不存在的主体、动作或场景描述，后续裁判就可能基于错误证据进行打标，导致误报。

第四，直接视频评估缺乏可追溯证据链。端到端视频问答模型往往直接给出结论，难以明确说明某一标签来自哪一帧、哪一片段或哪一处可见细节。对于视频生成质量评估而言，仅有最终标签是不够的，还需要知道标签对应的证据来源、错因描述和可复核位置。否则，即使模型结论偶然正确，也难以用于人工复审、错误分析和系统迭代。

第五，视频质量评估任务具有强标签边界要求。通用视频理解模型更擅长回答“视频里发生了什么”，但视频生成评估还要求模型判断“这个现象应该归入哪个质量标签”。这要求模型在提示词一致性、视觉缺陷、时间一致性、物理真实性、音频质量、审美质量和安全合规等标签之间进行精确归因。由于不同标签之间存在语义重叠，直接端到端评估容易出现标签混淆和重复归因。

基于上述问题，本项目没有直接采用“视频输入-标签输出”的端到端方式，而是将多模态视频解析拆分为可审计的分层证据生成过程。宏观层、片段层和逐帧层分别承担不同粒度的事实记录职责，再由裁判模型在文本证据空间中完成标签归因。这样既保留了多模态模型的视频理解能力，也降低了幻觉、遗漏和不可追溯带来的评估风险。

## 2. 方法框架

项目整体框架可以概括为“视频理解层”和“标签裁判层”两个阶段。

第一阶段是视频理解层。该阶段不直接判断视频是否正确，也不输出错误标签，而是根据不同粒度生成事实描述。当前系统将视频描述拆分为三个层级：

1. 宏观层：描述全片范围内稳定成立的主体、场景、风格、事件、动作、音频和整体呈现。
2. 片段层：按时间轴切分视频，描述不同片段中的主体延续、场景变化、动作过程、事件顺序、接触关系和音画关系。
3. 逐帧层：对关键帧或高频采样帧进行细节记录，描述局部结构、画面文字、图像质量、可见缺陷以及相邻帧变化。

第二阶段是标签裁判层。该阶段接收原始提示词、三层事实描述和评价标签体系，逐批判断视频是否命中某些问题标签。裁判层不直接依赖视频本身，而是依赖前一阶段生成的可追溯文本证据，从而降低视频多模态理解的不确定性，并增强标签结果的可解释性。

## 3. 数据流向

系统的数据流向如下：

1. 输入端包含原始生成提示词、待评估视频以及可能存在的参考素材；另一侧输入为结构化评价指标体系，即标签集合及其定义、正例和反例。
2. 视频分析模型接收视频和原始提示词，将视频转写为三层事实描述。描述时需要参考提示词中的核心要素，例如主体、属性、动作、场景、风格、文字和音频要求，但只记录“有、无、不可见、未知”等事实状态，不在分析层直接做对错判断。
3. 裁判模型接收三层文字描述、原始提示词和标签体系。系统按一级标签分批输入二级标签定义，并逐层传入宏观层、片段层和逐帧层描述，以降低上下文压力。
4. 裁判模型输出视频问题标签，同时保留每个标签对应的证据来源和错因描述。
5. 最终结果以结构化 JSON 形式保存，核心字段包括“一级标签--二级标签”“证据来源”和“错因描述”。

## 4. 关键过程问题与改进策略

### 4.1 标签归因错误

标签归因错误是当前系统召回率不足的重要原因之一。具体表现为：模型已经识别出视频中存在某类问题，但将该问题归入了错误标签。例如，视频中的人物没有执行提示词指定动作，模型能够发现“动作不符合要求”，但可能错误归因到“人物属性错误”。

该问题的根源在于标签边界不够清晰，或者模型对相近标签之间的判别标准掌握不足。尤其在“主体错误、属性错误、动作错误、场景错误”等提示词一致性标签之间，错误表现往往存在一定重叠，模型容易把同一现象归入语义相近但不精确的标签。

改进策略包括：

1. 在标签定义中加入正例和反例。正例用于说明某标签的典型适用场景，反例用于说明看似相近但不应归入该标签的情况，从而形成少样本边界约束。
2. 基于 MECE 原则重整标签体系，尽量减少标签之间的重叠，提高标签划分的互斥性和完备性。
3. 在裁判提示词中强调“先描述错因，再选择标签”，要求模型先明确错误现象，再判断最贴切的标签，而不是直接从标签名出发进行匹配。

### 4.2 上下文长度压力

完整输入通常包括原始提示词、三层视频描述、完整标签体系和裁判系统提示词。若一次性全部输入，容易造成上下文过长，导致模型报错、遗忘前文信息或忽略部分标签定义。

为缓解该问题，当前系统采用分批裁判策略：

1. 原始提示词不以无差别全文匹配为目标，而是在分析阶段重点提取其中的核心要素。
2. 标签体系不一次性全量灌入，而是按一级标签分批输入，每次只关注当前一级标签下的二级标签。
3. 三层描述可以逐层输入，避免宏观层、片段层和逐帧层同时占用过多上下文。
4. 通过“三层权责文档”明确不同描述层级的关注重点，使宏观层、片段层和逐帧层分别承担不同证据职责。一个层级可以服务多个标签，一个标签也可以由多个层级从不同角度提供证据。

### 4.3 反复归因与重复打标

模型反复归因是当前系统准确率较低的重要原因之一。某些错误表现并不能非常明确地归入单一标签，模型可能将同一问题重复归入多个标签，造成误报数量增加。例如，同一段主体外观变化既可能被归入属性错误，也可能被归入角色突变或局部生成失败。

为降低重复打标，系统在分批裁判过程中会将已有打标结果和打标理由继续传递给后续批次。后续裁判需要判断当前错因是否已经被已有标签覆盖，并在“新增、替换、保留、移除、并列”之间做出选择。原则上尽量避免并列，只有当同一现象确实对应独立错因时才允许保留多个标签。

该策略的目标是提高标签结果的精确性，使最终输出更接近人工标注中的“最主要问题归因”，而不是将一个错误现象泛化为多个相似标签。

### 4.4 项目迁移性不足

当前系统仍然与视频生成质量评估场景存在一定耦合。一旦迁移到其他视频评测任务，例如广告视频评测、影视镜头质量评测、教学视频质量评测或安全审核任务，可能需要重新设计标签体系、提示词结构和三层描述权责。

该问题当前已解决，我们提高了项目的模块化程度，将提示词、标签体系、裁判逻辑和数据看板解耦，使其成为可插拔组件。迁移到新项目时只需要替换标签体系和三层分析提示词，而不需要大规模重构整体流程。

## 5. 当前项目的核心局限

### 5.1 主观维度标签判定能力较弱

当前系统对客观事实类标签的处理相对更稳定，例如主体、属性、动作、文字、结构、时间突变等。这些标签通常可以通过可观察证据进行判断。

相比之下，涉及审美、灵动感、氛围、观看体验等主观维度的标签判定仍然较弱。此类判断高度依赖审美标准、任务语境和人工偏好，很难仅通过提示词工程显著提升一致性。

可能的改进方向包括引入人工标注数据进行模型微调，或构建更细粒度的审美评价量表。但这类方法成本较高，当前阶段暂不作为优先方向。

### 5.2 逐帧采样仍可能遗漏细粒度画面错误

当前系统中的“逐帧层”并不意味着对视频进行真正意义上的全帧穷举分析。以常见的 24 FPS 视频为例，若要完整逐帧检查，则每秒需要处理 24 帧，这会显著增加视觉 token 数量、模型调用成本和上下文压力。当前使用的视频理解模型在默认参数下通常更接近按秒级抽帧进行分析，即使提高采样参数，也仍然难以覆盖所有瞬时画面细节。

因此，逐帧层本质上应理解为“高频采样帧描述”或“关键帧级感知证据”，而不是对视频每一原始帧的完全遍历。这会带来一个局限：某些只在极短时间内出现的局部错误，例如手部瞬间畸变、物体短暂穿模、角色五官闪烁、文字一帧错误、动作中间态断裂等，仍可能在抽帧过程中被遗漏。

该问题的解决方向不是简单地对全视频盲目提高采样率，因为这会造成成本和上下文的快速膨胀。更可行的方式是采用自适应采样策略：

1. 优先选择画面变化显著的帧，例如镜头切换点、主体姿态突变点、场景亮度或结构快速变化点。
2. 优先选择时间不连贯或语义不稳定的片段，例如主体外观变化、物体突然出现或消失、动作阶段跳变等区域。
3. 对宏观层或片段层已经提示存在异常的时间段进行二次加密采样，而不是对整段视频均匀提高帧率。
4. 将关键帧检测与裁判结果联动：当裁判模型给出低置信度标签或证据不足判断时，回到对应时间段进行补充采样和复核。

通过这种“均匀采样 + 变化点采样 + 异常片段加密采样”的组合方式，可以在不显著增加整体成本的前提下，提高细粒度画面错误的发现概率。

## 6. 当前阶段总结

本项目当前形成了一套可运行的分层视频质量评估链路：通过宏观层、片段层和逐帧层将视频内容转化为文本证据，再由裁判模型结合原始提示词和标签体系生成结构化问题标签。

该方案的核心价值在于提高评估过程的可追溯性和可解释性。相比直接让模型对视频进行端到端打分，分层证据机制能够更清楚地展示某个标签对应的视频证据来源，也便于后续对错误归因、漏召回和重复打标问题进行分析。

当前系统仍需继续优化标签边界、裁判去重机制和主观维度评估能力，但整体框架已经能够支持批量视频评估、标签级准确率和召回率分析，以及后续人工审查和模型迭代。
